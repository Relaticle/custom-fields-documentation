</Warning>

## Overview

Custom Fields v2 brings significant improvements and new features while maintaining backward compatibility with v1. This major release focuses on enhanced flexibility, better performance, and more powerful field management capabilities.

## Key New Features

### 🎯 Visibility Rules

The most requested feature is finally here! Control when fields are displayed based on other field values.

<Card title="Visibility Rules" icon="eye" href="/v2/visibility-rules">
  Create conditional logic to show/hide fields dynamically
</Card>

**What you can do:**
- Show fields only when specific conditions are met
- Create dependent field relationships
- Build dynamic forms that adapt to user input
- Support for multiple conditions (AND/OR logic)

### 🚀 Performance Improvements

- **Optimized queries** - Reduced database queries by up to 40%
- **Lazy loading** - Fields load only when needed
- **Better caching** - Improved cache strategies for faster responses

### 🔧 Developer Experience

- **Improved API** - More intuitive and consistent API
- **Better TypeScript support** - Enhanced type definitions
- **Extended hooks** - More lifecycle hooks for customization

## Backward Compatibility

<Info>
All v1 features continue to work in v2. You can upgrade safely without breaking changes.
</Info>

- Existing custom fields work without modification
- Current API methods remain available
- Database structure is compatible
- License keys continue to work
