## Creating Custom Field Types

You can extend Custom Fields by creating your own field types. This is useful when you need specialized functionality not covered by the built-in types.

### Step 1: Create Your Field Type Class

Create a class that implements `FieldTypeDefinitionInterface`:

```php
namespace App\CustomFields\FieldTypes;

use Relaticle\CustomFields\Contracts\FieldTypeDefinitionInterface;
use Relaticle\CustomFields\Enums\FieldDataType;
use Relaticle\CustomFields\Enums\ValidationRule;
use Relaticle\CustomFields\FieldTypes\Concerns\HasCommonFieldProperties;

class StarRatingFieldType implements FieldTypeDefinitionInterface
{
    use HasCommonFieldProperties; // Provides common implementations
    
    public function getKey(): string
    {
        return 'star-rating';
    }
    
    public function getLabel(): string
    {
        return 'Star Rating';
    }
    
    public function getIcon(): string
    {
        return 'mdi-star';
    }
    
    public function getDataType(): FieldDataType
    {
        return FieldDataType::INTEGER;
    }
    
    public function getPriority(): int
    {
        return 50; // Lower numbers appear first
    }
    
    public function getFormComponentClass(): string
    {
        return StarRatingFormComponent::class;
    }
    
    public function getTableColumnClass(): string
    {
        return StarRatingColumn::class;
    }
    
    public function getInfolistEntryClass(): string
    {
        return StarRatingEntry::class;
    }
    
    public function allowedValidationRules(): array
    {
        return [
            ValidationRule::REQUIRED,
            ValidationRule::MIN,
            ValidationRule::MAX,
        ];
    }
}
```

### Step 2: Create Form Component

Create a Filament form component for your field type:

```php
namespace App\CustomFields\Components\Forms;

use Relaticle\CustomFields\Filament\Integration\Base\AbstractFormComponent;
use Filament\Forms\Components\Field;
use Filament\Forms\Components\Select;

class StarRatingFormComponent extends AbstractFormComponent
{
    protected function createField(): Field
    {
        return Select::make($this->getStatePath())
            ->options([
                1 => '⭐',
                2 => '⭐⭐',
                3 => '⭐⭐⭐',
                4 => '⭐⭐⭐⭐',
                5 => '⭐⭐⭐⭐⭐',
            ])
            ->native(false);
    }
}
```

### Step 3: Create Table Column

Create a table column for displaying your field:

```php
namespace App\CustomFields\Components\Tables;

use Relaticle\CustomFields\Filament\Integration\Base\AbstractColumn;
use Filament\Tables\Columns\Column;
use Filament\Tables\Columns\TextColumn;

class StarRatingColumn extends AbstractColumn
{
    protected function createColumn(): Column
    {
        return TextColumn::make($this->getStatePath())
            ->formatStateUsing(fn ($state) => str_repeat('⭐', (int) $state))
            ->html();
    }
}
```

### Step 4: Create Infolist Entry

Create an infolist entry for viewing your field:

```php
namespace App\CustomFields\Components\Infolists;

use Relaticle\CustomFields\Filament\Integration\Base\AbstractInfolistsComponent;
use Filament\Infolists\Components\Entry;
use Filament\Infolists\Components\TextEntry;

class StarRatingEntry extends AbstractInfolistsComponent
{
    protected function createEntry(): Entry
    {
        return TextEntry::make($this->getStatePath())
            ->formatStateUsing(fn ($state) => str_repeat('⭐', (int) $state))
            ->html();
    }
}
```

### Step 5: Register Your Field Type

Register your custom field type in a service provider:

```php
namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Relaticle\CustomFields\Facades\CustomFieldsType;
use App\CustomFields\FieldTypes\StarRatingFieldType;

class AppServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        CustomFieldsType::register([
            StarRatingFieldType::class,
        ]);
    }
}
```

## Advanced Field Type Features

### Supporting Options

If your field type needs predefined options (like select or radio), implement the option-related interfaces:

```php
use Relaticle\CustomFields\Contracts\HasOptionsInterface;

class CustomSelectFieldType implements FieldTypeDefinitionInterface, HasOptionsInterface
{
    // ... other methods
    
    public function requiresOptions(): bool
    {
        return true;
    }
    
    public function acceptsArbitraryValues(): bool
    {
        return false; // Set to true for tags-like behavior
    }
}
```

### Custom Data Types

The available data types determine how values are stored:

```php
enum FieldDataType: string
{
    case TEXT = 'text';
    case INTEGER = 'integer';
    case FLOAT = 'float';
    case BOOLEAN = 'boolean';
    case DATE = 'date';
    case DATETIME = 'datetime';
    case JSON = 'json';
}
```

### Encryption Support

Enable encryption for sensitive field types:

```php
public function isEncryptable(): bool
{
    return true; // Allows users to encrypt this field's values
}
```

### Search and Filter Support

Control table functionality:

```php
public function isSearchable(): bool
{
    return true; // Enable text search
}

public function isSortable(): bool
{
    return true; // Enable column sorting
}

public function isFilterable(): bool
{
    return true; // Enable filtering
}

public function getTableFilterClass(): ?string
{
    return StarRatingFilter::class; // Custom filter component
}
```

## Using the HasCommonFieldProperties Trait

The `HasCommonFieldProperties` trait provides default implementations for common methods:

```php
use HasCommonFieldProperties;

// Provides defaults for:
// - getTableFilterClass() => null
// - isSearchable() => true
// - isSortable() => true
// - isFilterable() => false
// - isEncryptable() => false
// - acceptsArbitraryValues() => false
```

## Field Type Priority

Field types are ordered by priority (lower numbers first):

- **10-20**: Common text fields (text, number, textarea)
- **30-40**: Selection fields (select, radio, checkbox)
- **50-60**: Specialized fields (date, color, currency)
- **70+**: Advanced fields (rich editor, markdown)

## Best Practices

1. **Reuse Existing Components**: Extend Filament's built-in components when possible
2. **Follow Naming Conventions**: Use descriptive keys like `company-lookup` or `rating-stars`
3. **Validate Appropriately**: Only allow validation rules that make sense for your field type
4. **Document Your Field**: Add PHPDoc comments explaining the field's purpose
5. **Test Thoroughly**: Create tests for form input, table display, and data storage

## Example: Country Select Field

Here's a complete example of a country selection field:

```php
// 1. Field Type Definition
class CountrySelectFieldType implements FieldTypeDefinitionInterface
{
    use HasCommonFieldProperties;
    
    public function getKey(): string
    {
        return 'country-select';
    }
    
    public function getLabel(): string
    {
        return 'Country Select';
    }
    
    public function getIcon(): string
    {
        return 'mdi-earth';
    }
    
    public function getDataType(): FieldDataType
    {
        return FieldDataType::TEXT;
    }
    
    public function getFormComponentClass(): string
    {
        return CountrySelectFormComponent::class;
    }
    
    // ... other required methods
}

// 2. Form Component with country list
class CountrySelectFormComponent extends AbstractFormComponent
{
    protected function createField(): Field
    {
        return Select::make($this->getStatePath())
            ->options(Countries::all()->pluck('name', 'code'))
            ->searchable()
            ->preload();
    }
}

// 3. Register in AppServiceProvider
CustomFieldsType::register([
    CountrySelectFieldType::class,
]);
```

Your custom field type will now appear in the field type dropdown when creating new custom fields!
