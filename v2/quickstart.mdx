## Step 1: Create Your Product Model (2 minutes)

First, let's create a Product model if you don't have one:

```bash
php artisan make:model Product -m
```

Update the migration:

```php
// database/migrations/xxxx_create_products_table.php
Schema::create('products', function (Blueprint $table) {
    $table->id();
    $table->string('name');
    $table->text('description')->nullable();
    $table->decimal('price', 10, 2);
    $table->string('status')->default('active');
    $table->timestamps();
});
```

Configure the model for custom fields:

```php
// app/Models/Product.php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Relaticle\CustomFields\Models\Contracts\HasCustomFields;
use Relaticle\CustomFields\Models\Concerns\UsesCustomFields;

class Product extends Model implements HasCustomFields
{
    use UsesCustomFields;
    
    protected $fillable = [
        'name',
        'description', 
        'price',
        'status',
    ];
}
```

Run the migration:
```bash
php artisan migrate
```

## Step 2: Create Filament Resource (3 minutes)

Generate a Filament resource for products:

```bash
php artisan make:filament-resource Product --generate
```

The Entity Management System will automatically discover your Product model! No additional configuration needed.

Verify entity registration:
```bash
php artisan custom-fields:list-entities
```

You should see:
```
Registered Entities:
- App\Models\Product (Product)
```

## Step 3: Add Your First Custom Fields (4 minutes)

Navigate to your admin panel and go to **Custom Fields → Manage Fields**.

### Create a Product Category Field

<Steps>
  <Step title="Select Entity">
    Choose "Product" from the entity dropdown
  </Step>
  
  <Step title="Add Field">
    Click "New Custom Field" and fill in:
    - **Name**: Product Category
    - **Type**: Select
    - **Options**: Electronics, Clothing, Books, Home & Garden
    - **Required**: Yes
  </Step>
  
  <Step title="Save">
    Click "Create" to save the field
  </Step>
</Steps>

### Add More Fields

Quickly add these additional fields:

1. **SKU Field**
   - Name: SKU
   - Type: Text
   - Validation: Required, Unique
   - Placeholder: "PROD-001"

2. **Weight Field**
   - Name: Weight (kg)
   - Type: Number
   - Validation: Required, Min: 0.01
   - Step: 0.01

3. **In Stock Toggle**
   - Name: In Stock
   - Type: Toggle
   - Default: true

## Step 4: See Custom Fields in Action (3 minutes)

Go to **Products** in your admin panel and create a new product:

1. Fill in the basic fields (Name, Description, Price)
2. Notice the "Custom Fields" section automatically appears
3. Fill in your custom fields:
   - Select "Electronics" for category
   - Enter "ELEC-001" for SKU
   - Set weight to "0.5"
   - Toggle "In Stock" on

Save the product and see all your custom data stored!

## Step 5: Add Conditional Fields (2 minutes)

Let's add fields that only show for electronic products:

### Create Warranty Field

1. Go to **Custom Fields → Manage Fields**
2. Add new field:
   - **Name**: Warranty Period (months)
   - **Type**: Number
   - **Visibility**: Conditional

3. Set visibility condition:
   ```
   Field: Product Category
   Operator: Equals
   Value: Electronics
   ```

### Create Battery Type Field

Add another conditional field:
- **Name**: Battery Type
- **Type**: Select
- **Options**: Lithium-ion, AA, AAA, Built-in, N/A
- **Visibility**: Same condition as warranty

Now when you edit a product and select "Electronics" as the category, these fields appear automatically!

## Step 6: Quick Import Test (1 minute)

### Generate Import Template

1. Go to **Custom Fields → Import/Export**
2. Select "Product" entity
3. Click "Download Template"

### Prepare Sample Data

Edit the CSV template with sample products:

```csv
name,description,price,status,custom_field_sku,custom_field_weight_kg,custom_field_product_category
"Laptop Pro",,"1299.99","active","ELEC-002","1.2","Electronics"
"Winter Jacket",,"89.99","active","CLOTH-001","0.8","Clothing"
"Coffee Table Book",,"34.99","active","BOOK-001","2.1","Books"
```

### Import Products

1. Upload the CSV file
2. Review field mapping
3. Click "Import"

Check your products list - all imported with custom fields! 🎉

## Troubleshooting

<Accordion title="Fields not showing in forms?">
  1. Clear cache: `php artisan cache:clear`
  2. Verify entity registration: `php artisan custom-fields:list-entities`
  3. Check model implements `HasCustomFields` interface
</Accordion>

<Accordion title="Import not working?">
  1. Ensure CSV headers match: `custom_field_` + field slug
  2. Check date formats match your configuration
  3. Verify required fields have values
</Accordion>

<Accordion title="Conditional fields not hiding/showing?">
  1. Ensure parent field has a value
  2. Check operator matches (equals, not_equals, contains)
  3. Verify field slugs in conditions
</Accordion>
