This guide will have you adding custom fields to your Filament resources in minutes using the new v2 API.

<Note>
Make sure you've completed the [Installation](/v2/installation) steps first.
</Note>

## Step 1: Register the Plugin

In your `AdminPanelProvider` or panel configuration, register the Custom Fields plugin:

```php
use Relaticle\CustomFields\CustomFieldsPlugin;
use Filament\Panel;

public function panel(Panel $panel): Panel
{
    return $panel
        // ... other panel configurations
        ->plugins([
            CustomFieldsPlugin::make(),
        ]);
}
```

## Step 2: Prepare Your Model

Your model needs to implement the `HasCustomFields` interface and use the `UsesCustomFields` trait.

```php
namespace App\Models\Shop;

use Illuminate\Database\Eloquent\Model;
use Relaticle\CustomFields\Models\Concerns\UsesCustomFields;
use Relaticle\CustomFields\Models\Contracts\HasCustomFields;

class Customer extends Model implements HasCustomFields
{
    use UsesCustomFields;

    // ... your existing model code
}
```

## Step 3: Add Custom Fields to Forms (v2 Syntax)

Custom Fields v2 introduces a fluent builder pattern for forms. Add custom fields to your resource forms using the new API:

```php
use Relaticle\CustomFields\Facades\CustomFields;
use Filament\Schemas\Schema;
use Filament\Forms;

class CustomerResource extends Resource
{
    public static function form(Schema $schema): Schema
    {
        return $schema
            ->components([
                // Your existing form fields
                Forms\Components\TextInput::make('name')
                    ->required(),
                Forms\Components\TextInput::make('email')
                    ->email()
                    ->required(),

                // Add Custom Fields using v2 builder pattern
                CustomFields::form()
                    ->forModel($schema->getRecord())
                    ->build()
                    ->columnSpanFull()
            ]);
    }
}
```

<Info>
The v2 API uses `CustomFields::form()->forModel()->build()` instead of the direct component approach, providing better type safety and IDE support.
</Info>

## Step 4: Display Custom Fields in Tables (v2 Syntax)

Add custom field columns to your table using the new spread operator syntax:

```php
use Relaticle\CustomFields\Facades\CustomFields;
use Filament\Tables\Table;
use Filament\Tables;

public static function table(Table $table): Table
{
    return $table
        ->columns([
            Tables\Columns\TextColumn::make('name')
                ->searchable(),
            Tables\Columns\TextColumn::make('email')
                ->searchable(),
            
            // Spread custom field columns
            ...CustomFields::table()
                ->forModel($table->getModel())
                ->columns(),
        ])
        ->filters([
            // Add custom field filters
            ...CustomFields::table()
                ->forModel($table->getModel())
                ->filters(),
        ]);
}
```

## Step 5: Add Custom Fields to Info Lists (v2)

For view pages, include custom fields in your infolist:

```php
use Relaticle\CustomFields\Facades\CustomFields;
use Filament\Schemas\Schema;
use Filament\Infolists;

public static function infolist(Schema $schema): Schema
{
    return $schema
        ->components([
            Infolists\Components\Section::make()
                ->schema([
                    Infolists\Components\TextEntry::make('name'),
                    Infolists\Components\TextEntry::make('email'),
                ])
                ->columns(2),

            // Custom Fields section
            CustomFields::infolist()
                ->forModel($schema->getRecord())
                ->build(),
        ]);
}
```

## Step 6: Custom Field Sections (New in v2)

Version 2 introduces Custom Field Sections for better organization:

```php
// Custom fields are automatically grouped into sections
// defined in the database migration
CustomFields::form()
    ->forModel($schema->getRecord())
    ->withSections() // Enable section grouping
    ->build()
```

## Working with Custom Field Values

v2 provides improved methods for accessing custom field values:

```php
// Get a custom field value
$customer->customField('field_code');

// Set a custom field value
$customer->setCustomField('field_code', 'value');

// Get all custom fields
$customer->customFields();

// Check if a custom field exists
$customer->hasCustomField('field_code');
```

## Import/Export Support (v2)

v2 includes built-in import/export support:

```php
use App\Filament\Imports\Shop\CustomerImporter;

class CustomerImporter extends Importer
{
    protected static ?string $model = Customer::class;

    public static function getColumns(): array
    {
        return [
            ImportColumn::make('name')
                ->requiredMapping()
                ->rules(['required']),
            ImportColumn::make('email')
                ->requiredMapping()
                ->rules(['required', 'email']),
            
            // Auto-include custom fields
            ...CustomFields::import()
                ->forModel(static::getModel())
                ->columns(),
        ];
    }
}
```

## Performance Optimizations in v2

Version 2 includes significant performance improvements:

- **Eager loading**: Custom fields are automatically eager-loaded
- **Caching**: Field definitions are cached for better performance
- **Optimized queries**: Reduced N+1 query problems
- **Lazy loading**: Fields load only when needed
